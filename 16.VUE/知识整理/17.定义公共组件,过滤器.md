### Vue.use

```js
Vue.use( plugin )
参数：{Object | Function} plugin
用法：
安装 Vue.js 插件。如果插件是一个对象，必须提供 install 方法。如果插件是一个函数，它会被作为 install 方法。install 方法调用时，会`将 Vue 作为参数传入`。

该方法需要在调用 `new Vue() 之前`被调用。

当 install 方法被同一个插件多次调用，插件将只会被安装一次。
```

### 管理全局组件,指令,过滤器

```js
我们需要模块化的管理全局的组件,指令,过滤器,以及挂载在vue全局实例的方法.
这个时候,我们先创建一个`core`的文件夹.在core里面创建一个global的文件夹和一个core.config.js的文件,
    global文件夹用来存放组件,过滤器等等.core.config.js是配置全局组件,过滤器的js文件
```

在webpack打包文件的入口文件main.js文件里面:

```js
//导入core.config.js
import core from "src/core/core.config.js"
Vue.use(core)
//为什么使用Vue.use(core)呢,因为Vue.use()可以为插件传入Vue实例,相当于this,vue实例对象.
```

在core.config里面:

```js
export default (Vue)=>{    // 在这里接收Vue.use()传入的Vue实例对象
    Vue.directive("test",(el,binding,vnode){   //全局指令
           el.onclick = function(){
        	alert("ok")
    	}
     })
}
就这样,我们暴露了一个全局指令,怎么暴露的呢?
    首先,我们在core.config.js里面使用Vue.directive定义全局指令,当是,这个Vue全局指的是传如的Vue对象;
然后,我们在main.js导入core.config.js,使用Vue.use(core)来为core插件传入当前的Vue实例对象.
```
